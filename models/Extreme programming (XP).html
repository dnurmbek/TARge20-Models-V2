Extreme programming (XP)

XP: Advantages and Disadvantages

Advantages Disadvantages
☆ This methodology allows software development companies to save costs and time required for project realization. ★ It is said that Extreme Programming is focused on the code rather than on design.
☆ Simplicity is one more advantage of Extreme Programming projects. The developers who prefer to use this methodology create extremely simple code that can be improved at any moment. ★ In XP projects the defect documentation is not always good.
☆ The whole process in XP is visible and accountable. Developers commit what they will accomplish and show progress. ★ XP is not the best option if programmers are separated geographically.
